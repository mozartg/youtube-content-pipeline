# UPWORK OPERATING DOCTRINE — v1 (2026-06-24)
Owner: Cowork (Claude). Supersedes the loose "scan + draft 5" worker-floor Upwork recipe.
Read this BEFORE every Upwork run. Check every bid against it.

## 0. The reframe (Mo's correction, 2026-06-24)
Upwork is **competitive, not messy.** Winning is not volume — it's **speed + scoping + building + planning**, then **Mo pulls the trigger.** Pray-and-spray fails and burns money. We bid few, bid sharp, and bring proof.

## 1. CONNECTS = REAL MONEY (budget discipline)
- 1 Connect = **$0.15**. Submitting costs **2–16 Connects** per job (set by Upwork; Expert/featured = 16). Boosting adds 6–40+.
- Mo's balance is small (was **81 Connects**). At 16/bid that's ~5 expensive bids. Treat Connects like cash.
- **Rule:** never spend Connects on a job scoring < 0.45 win-probability (see §3) unless fit is exceptional AND it's a one-shot floor-clear. Log Connects spent per bid in the tracker.
- **Boost only** when win-prob ≥ 0.6 AND budget clears the floor in one win. Otherwise no boost.
- Target: **≤ 3 high-conviction bids per run**, not 5+. Quality of bid > number of bids.

## 2. NO-SKIM RULE (hard)
Read the **entire** job post, **all attachments**, and **every screening question** before drafting. Quote the client's own words. If the post names a tool, deliverable, or constraint, the proposal must address it specifically. Skimming = an instant tell and a wasted Connect. (Past miss: invented a "morning review dashboard" the client never asked for; mislabeled the delivery method. Never again — mirror the actual scope.)

## 3. WIN-PROBABILITY SCORE (from public Upwork signals)
Read these off the job page every time and score 0–1:
- **Proposals (bids):** <5 = strong (+), 5–10 = good, 10–15 = moderate, 15–20 = competitive (−), 20–50 = crowded (−−), 50+ = avoid.
- **Hires:** if **Hires ≥ 1** on a one-time job → treat as ~filled (kill). (Verified 2026-06-24: a 3-day-old job already had 1 hire.)
- **Interviewing:** 0–2 ok; **5+ = client already shortlisting**, cold bids drop hard.
- **Invites sent / unanswered:** many invites = client courting specific people; cold applicant odds fall.
- **Client:** payment-verified + real spend + high hire-rate = can/will pay. Unverified/$0 = discount probability, ask for a verified first milestone.
- **Recency / last-viewed:** apply **same day**; jobs decay in 3–5 days. If client hasn't viewed in days, deprioritize.
- **Fit to Mo's true record** (nonprofit/DEI/social work, CRM/ops, automation): multiplier.
Rough score = signal blend. **Apply only ≥ 0.45; boost only ≥ 0.6.**

## 4. SAME-DAY RULE
A job is only worth a Connect the day it posts (ideally within hours). Any job carried > 48h unsubmitted is auto-dead — close its ClickUp task, don't re-pitch a stale post. Freshness is the cheapest edge.

## 5. BUILD & SHIP — bring proof before Mo triggers
Speed → scope → BUILD → plan → Mo pulls the trigger. Where proof lifts win-prob, ship a real artifact attached to the proposal:
- **Canva** → one-page proposal visual, case-study deck, or portfolio leave-behind. Drop a **shareable link** in the proposal.
- **Kimi = Vercel** → when a job wants a working thing (mockup site, mini-app, dashboard), have Kimi deploy a live URL to show, not describe.
- **GitHub** (Kimi-built) → repos that demonstrate automation/QA rigor; link in "GitHub/website" fields.
- **Cowork (here)** → decks (pptx), sample-workflow one-pagers, scoping docs, trackers, scripts.
- **Desktop Commander** → local .pdf/.docx/.xlsx deliverables + read Mo's real files for proof.
- **PDF viewer** → verify any generated PDF before Mo sends.
Rule: a proposal that *shows* (link/sample) beats five that *tell*. One proof artifact per high-conviction bid.

## 6. CAPABILITY ROUTING (who does what — 2026-06-24)
- **Cowork (Claude)** = scope, build artifacts, govern, verify, drive ClickUp/Drive/Calendar/Canva.
- **Kimi** = Vercel deploys (live sites/apps), GitHub repo builds, cross-assistant harvesting, volume.
- **Claude Code** = WHEN LIVE, the primary execution engine for file I/O, builds, deploys, scheduled work. Route heavy build there; Cowork reviews/decides.
- **Canva** = decks + shareable links + portfolio/leave-behinds.
- **Mo** = last-mile only: review, submit, pull the trigger. NEVER auto-submit a proposal or spend Connects without Mo.

## 7. PER-JOB OUTPUT CONTRACT
For each high-conviction job, produce: (1) full-read scope summary in the client's words, (2) tailored proposal — concrete deliverable first, priced to band, 3 credibility bullets on Mo's real record, one question, (3) one proof artifact + link where it lifts odds, (4) win-prob score + Connects cost + boost yes/no, (5) screening-question answers if any, (6) one last-mile line for Mo.

## 8. THE LOOP
1. Pull fresh jobs (same-day). 2. Read full posts + attachments + screening Qs (no skim). 3. Score win-prob; drop < 0.45. 4. For survivors (≤3): scope → build proof → draft proposal + answers → compute Connects. 5. Stage in ClickUp + tracker. 6. Mo reviews and submits. 7. Close stale/dead tasks; log Connects + outcomes for calibration.

## v1.1 FIELD LESSONS (appended 2026-06-26 by the CAIRN thread — append-only, do not overwrite v1)
- **NO-SKIM caught a dead job.** Reading the full job PAGE (not the feed snippet) surfaced a client whose account was
  SUSPENDED ("Apply" disabled). Always open the page; the feed will let you waste Connects on an unbiddable job.
- **Pull bid cost + signals from the detail page.** Connects cost, proposals, interviewing, "last viewed", payment-verified,
  and avg-hourly-paid live on the job detail page only, and they move win-probability (a $228K buyer paying $5.47/hr avg is a
  low-rate buyer despite looking elite).
- **Flag gaps as either/or, don't bury or overclaim.** Best drafts this run each carried one explicit flag (full-time hours vs
  Mo's PT-only; mandatory Zoho Creator skill gap; unverified billing). Honesty is the differentiator.
- **Prioritize AI-naive low-spec buyers** (inbox cleanup, CRM/record cleanup) — operator-who-AI-leverages out-delivers per dollar
  vs crowded commodity-dev races.
